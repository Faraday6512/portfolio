Project Hello Quad

3D Graphics

Project Overview

Hello Quad is the first assignment in CS250 Computer Graphics II at DigiPen Institute of Technology. The goal of this project was to build a foundational OpenGL rendering pipeline from scratch by implementing core GPU resource management classes and writing animated GLSL shaders. The project targets OpenGL 3.3, OpenGL ES 3.0, and WebGL 2, running on both Windows and Web platforms.

Tasks Completed

  • Implemented VertexBuffer — GPU vertex buffer creation, data upload, and lifecycle management using OpenGL buffer objects.
  • Implemented IndexBuffer — Index buffer creation and binding for indexed draw calls.
  • Implemented VertexArray — VAO setup with vertex attribute pointer configuration, supporting multiple buffer layouts and instancing divisors.
  • Implemented Texture — 2D texture loading from image files and memory, with configurable filtering and wrapping modes.
  • Wrote a vertex shader that rotates the quad over time and moves it to follow the mouse cursor.
  • Wrote a fragment shader that smoothly cycles between vertex colors and texture colors using a time-based sin function.
  • Used window::ElapsedTime and window::input::MouseWindowX/Y environment variables to drive animations.
  • Built and deployed a WebGL 2 version using Emscripten.

Reflection

The most challenging part was implementing the VertexArray class, specifically setting up vertex attribute pointers with the correct stride and offset calculations across multiple attributes. Understanding the difference between glVertexAttribPointer and glVertexAttribIPointer and when to use each was an important lesson. I also learned how the VAO stores the relationship between vertex buffers and attribute slots, and why the index buffer must remain bound when the VAO is unbound. Working with the asset reloader to hot-reload shaders at runtime was a great way to iterate quickly on the animations.
