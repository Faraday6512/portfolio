Project Procedural Geometric Modeling

3D Graphics

Project Overview

Procedural Geometric Modeling is the second assignment in CS250 Computer Graphics II at DigiPen Institute of Technology. The goal of this project was to implement a mesh generation system that procedurally builds 3D geometry entirely in code — without loading any external model files. Shapes such as planes, cubes, spheres, cylinders, and cones are constructed by computing vertex positions, normals, UV coordinates, and index buffers from scratch. The project targets OpenGL 3.3, OpenGL ES 3.0, and WebGL 2, running on both Windows and Web platforms.

Tasks Completed

  • Implemented a Mesh data structure to hold vertex attribute arrays (position, normal, UV, color) and an index list.
  • Procedurally generated a Plane mesh with configurable subdivisions, computing normals and UV coordinates per vertex.
  • Procedurally generated a Cube mesh with correct per-face normals and UV mapping for each of the six faces.
  • Procedurally generated a Sphere mesh using a UV-sphere algorithm with stacks and slices, including smooth per-vertex normals.
  • Procedurally generated a Cylinder and Cone mesh with capped ends and correctly interpolated side normals.
  • Uploaded all generated mesh data to the GPU using the VertexBuffer, IndexBuffer, and VertexArray classes from the previous assignment.
  • Rendered multiple mesh instances in a scene with different transforms using a simple Phong-style shader.
  • Built and deployed a WebGL 2 version using Emscripten.

Reflection

The most challenging part was correctly computing smooth normals for the sphere — making sure each vertex normal pointed outward along the unit sphere surface rather than approximating it from face normals. Getting the index winding order consistent across all shapes so that back-face culling worked correctly also required careful attention. I learned how the same vertex data layout built in the previous assignment composes cleanly with any procedurally generated geometry, which reinforced why a well-designed buffer abstraction matters. Seeing complex shapes appear from pure math — without any mesh file involved — was a satisfying demonstration of how all 3D geometry is ultimately just arrays of numbers.
